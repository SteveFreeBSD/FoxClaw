# Security Response

## Intake

- Primary reporting path: `../SECURITY.md`.
- Private vulnerability reports should be submitted through GitHub Security Advisories.

## Triage

- Initial acknowledgement target: within 5 business days.
- Triage records must include severity, affected versions, and reproduction notes.

## Remediation

- High and critical issues require tracked fix plans and verification evidence.
- Release notes must identify patched issue scope and operator action required.

## Disclosure

- Coordinate disclosure timing after fix availability or agreed mitigation window.
- Update `../SECURITY.md` and relevant runbooks when process changes.
